The next step is setting up guest access. You may remember we created a media share datastore/media. We want to share that with guests to the network (on the trusted subnet anyway). Before we go ahead and set that up, we want to map the windows Guest user to the unix user nobody.

Now we‚Äôve got CIFS set up for our clients, I want to set up NFS shares for other Linux boxes on the network (at this point, only my router) to be able to access. Now, the idea is that a cron job will run rsyncing over the files every hour, on the hour.¬† For many reasons (in case I get a virus and need to revert back, in case somebody hacks in and does bad stuff, etc, etc) I‚Äôm going to choose to create Snapshots so I can roll back to a previous version.

The convention I want is hourly.HOUR, daily.DAY, weekly.WEEK for up to 7 days and 4 weeks.¬† This also means that once I delete a file, I won‚Äôt recover the space that it took (once a snapshot of the file has been created) in my data pool until the end of the 4 week period.¬† for instance, hourly.0 will be the last hours snapshot, hourly.1 will be the 2nd last hours snapshot, etc.

the following bash script will take care of the desired snapshots.¬† It‚Äôs based on a concept I took from this rolling snapshots made easy post but I like my scripted way of doing rotating snapshots much better.

#!/bin/bash

#print out usage

if [ $# -ne 2 ]

then

echo ‚Äù¬† eg. snapshot.sh hour 24‚Ä≥

#if the max snapshot already exists, just delete it
